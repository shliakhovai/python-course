# Технічне завдання

Необхідно реалізувати консольну програму для збору та валідації даних користувача.

Програма повинна:

- запитувати ім'я, пароль та email;
- перевіряти їх відповідно до заданих правил;
- застосовувати опціональні трансформації (маскування, шифрування, ініціали).

Завдання спрямоване не лише на синтаксичну коректність, а й на дисципліну роботи з помилками, перевірку введених даних та структуризацію коду.

> **Важливо:**
>
> - Весь текст у коді має бути **англійською мовою**.
> - Треба придумати гарне ім'я для файлу.
> - Не забувати про гарний **title** та **commit message** у Pull Request.

---

## Функціональні вимоги

## 1. Валідація імені

**Вхід:** рядок з повним ім'ям (ім'я + прізвище).

Вимоги:

- Ім'я повинно містити щонайменше **два слова**.
- Кожна частина повинна мати довжину **не менше 2 символів**.

За прапором `show_only_initials` (визначається з відповіді користувача `y/n`):

- `True` → повернути ініціали у форматі `N.S.`
- `False` → повернути нормалізоване повне ім'я.

---

## 2. Валідація пароля

Вимоги:

- Мінімальна довжина — **8 символів**.
- Має містити:
  - хоча б одну цифру;
  - хоча б одну велику літеру.

Якщо `encrypt_password` (визначається з відповіді користувача `y/n`):

- базова реалізація — реверс рядка;
- додаткове завдання — реалізувати нетривіальний шифр (наприклад, шифр Цезаря або XOR), який при цьому є оборотнім.

---

## 3. Валідація email

Вимоги:

- Мінімальна довжина — **15 символів**.
- Повинен містити **рівно один символ `@`**.
- Повинен закінчуватись на:
  - `.com`
  - `.org`
  - `.ua`

Якщо `mask_email` (визначається з відповіді користувача `y/n`):

- замаскувати локальну частину (між першою та останньою літерою замінити символи на `*`).

---

## 4. Взаємодія з користувачем

- Дані вводяться через `input()`.
- Потрібно обрізати пробіли (`strip()`).
- Нормалізувати формат введення (наприклад, привести ім'я до `Title Case`).
- Після введення кожного поля запитувати у користувача, чи застосовувати відповідну трансформацію (`y/n`).

### Приклад сесії

```text
Введіть ім'я та прізвище: john doe
Показати лише ініціали? (y/n): y
Ім'я: J.D.

Введіть пароль: MyPass1
Зашифрувати пароль? (y/n): y
Пароль збережено (зашифровано).

Введіть email: john.doe@example.com
Замаскувати email? (y/n): y
Email: j*******e@example.com
```

---

## 5. Вимоги до обробки помилок

У всіх функціях необхідно:

- використовувати `raise ValueError` з інформативними повідомленнями;
- не допускати завершення програми без пояснення причини помилки;
- забезпечити сувору валідацію всіх введених даних.

---

## Інструкція для студентів

- Використовувати зрозумілі назви змінних і функцій.
- Весь текст у коді (змінні, функції, повідомлення) — англійською мовою.
- Оформіть рішення у вигляді Pull Request (PR) і надайте мені посилання в особисті повідомлення.
- Не забуваємо про гарні commit messages при використанні команди:

```bash
git commit -m "..."
```
