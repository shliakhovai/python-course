# Домашнє завдання

## 1. Логування користувача

Ви розробляєте просту систему логування для внутрішнього сервісу.

Користувач вводить своє ім'я та прізвище в одному рядку (наприклад: `Іван Петренко`).

Система повинна:

- Перевірити, що користувач щось ввів (рядок не порожній і не складається лише з пробілів).
- Якщо введення коректне, сформувати ініціали.
- Якщо ні — вивести повідомлення про помилку.

### Приклад

```text
Введіть ваше повне ім'я через пробіл: Іван Петренко
Ініціали: І П
```

```text
Введіть ваше повне ім'я через пробіл:
Помилка: введіть реальне ім'я
```

## 2. Маскування email

У системі відображення профілю потрібно приховувати частину email з міркувань безпеки.

Користувач вводить свій email.

Система повинна:

- Залишити домен без змін.
- Замаскувати ім'я користувача (частину до `@`), залишивши тільки перший і останній символ.
- Всі інші символи замінити на `*`.

### Додатково

Можна додати перевірку:

- що рядок не порожній;
- що пошта закінчується тільки на `.com` або `.org` (використовувати зріз, а потім знайти готовий метод).

## 3. Додавання унікального значення (створи свій `set()`)

У вас є список вже зареєстрованих значень:

```python
[1, 2, 3, 4, 5]
```

Користувач вводить нове число.

Система повинна:

- Перевірити, чи таке значення вже існує.
- Якщо ні — додати його до списку.
- Якщо вже існує — залишити список без змін.

### Приклад

```text
Яке число ви хочете додати? 5
Ось як виглядає список до змін: [1, 2, 3, 4, 5]
Це число вже існує, залишаємо список як є: [1, 2, 3, 4, 5]
```

```text
Яке число ви хочете додати? 10
Ось як виглядає список до змін: [1, 2, 3, 4, 5]
Змінили список, тепер він виглядає так: [1, 2, 3, 4, 5, 10]
```

## 4. Аналіз тегів (`set` + `union` / `intersection`)

Ви працюєте над системою рекомендацій.

Два користувачі вводять свої інтереси у вигляді тегів через кому.

Кількість тегів обмежена — не більше трьох.

Потрібно:

- Створити дві пусті множини для тегів першого і другого користувача.
- Спитати у двох користувачів їхні інтереси.
- Перетворити введені рядки у список тегів.
- Додати теги у множини саме за допомогою `.update()`.
- Знайти спільні інтереси користувачів.
- Знайти всі унікальні інтереси обох користувачів.
- Вивести обидва результати.

### Підказка

- Зверніть увагу, що після коми користувач може вводити пробіли.
- Для того, щоб прибрати пробіли, можна використовувати `.strip()`.

### Приклад

```text
Теги користувача 1: python, ai, javascript
Теги користувача 2: ai, music, war

Спільні: {'ai'}
Унікальні: {'war', 'javascript', 'music', 'ai', 'python'}
```

```text
Теги користувача 1:
Теги користувача 2:

Помилка: введіть теги
```

### Додатково

Можна додати перевірки:

- що рядок не порожній;
- що користувач ввів не більше та не менше трьох тегів.

## 5. Обробка рядка з числами

Ви реалізуєте обробку введених даних у калькуляторі.

Користувач вводить рядок значень через пробіл.

Система повинна:

- Перевірити, чи всі значення є числами (розділити строку на список, знайти готовий метод для перевірки, чи є значення числом, перевірити, звертаючись за індексом).
- Якщо так, виведіть суму перших трьох чисел.
- Якщо ні — повідомити про помилку.

### Приклад

```text
Введіть, будь ласка, числа через пробіл: 5 10 15
Сума трьох чисел: 30
```

```text
Введіть, будь ласка, числа через пробіл: 5 10 yes
Помилка: одне значення або всі не є числами.
```

# Інструкція для студентів

- Працювати тільки в одному файлі, а завдання відокремлювати коментарями.
- Використовувати зрозумілі назви змінних.
- Дотримуйтесь охайного форматування коду.
- Якщо застосовується новий метод (з яким ви не працювали або його не було на заняттях), поясніть його коментарем.

### Як приклад

- Оформіть рішення у вигляді Pull Request (PR) і надайте мені посилання в особисті повідомлення.
- Не забуваємо про гарні commit messages при використанні команди:

```bash
git commit -m "..."
```
