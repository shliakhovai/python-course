# Домашнє завдання

## Pre-commit hooks

У цьому завданні ти налаштовуєш автоматичну перевірку коду перед кожним комітом за допомогою `pre-commit`.

---

## Кроки виконання

### 1. Додай файли конфігурації

З архіву, отриманого на занятті, додай до свого репозиторію з домашніми завданнями файли конфігурації pre-commit hooks.

### 2. Запусти перевірку всіх правил

```bash
pre-commit run --all-files
```

Ця команда перевіряє весь код у репозиторії за всіма налаштованими правилами — навіть без коміту.

### 3. Виправ помилки

Переглянь усі знайдені порушення та виправ їх у коді. Запускай перевірку повторно, доки всі правила не пройдуть.

### 4. Відкрий Pull Request

Оформи всі зміни як Pull Request. Він має включати:

- файли конфігурації pre-commit hooks;
- всі виправлення коду, яких вимагав linter.
